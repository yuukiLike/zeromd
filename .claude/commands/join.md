新会话的冷启动入口。读取 `todo.md` 帮助用户（和 Claude）快速恢复上下文、进入开发状态。

## 步骤

### 1. 读取 todo.md

使用 Read 工具读取项目根目录的 `todo.md`。

如果文件不存在或为空，输出：

```
📭 没有待办事项。用 /goodgame 记录下次要做的事。
```

并结束。

### 2. 快速检查当前项目状态

```bash
git branch --show-current
git status --porcelain
git log -3 --oneline
```

与 todo.md 记录的状态对比，确认是否一致（比如 todo.md 说"在 feat/xxx 分支"，但现在在 main 上）。

### 3. 解析并展示

阅读 todo.md 的内容，用简洁的格式向用户展示：

- 上次的进度和当前状态
- 未完成的事项
- 待讨论的设计决策
- 下一步要做的功能或任务

### 4. 建议行动

根据 todo.md 的内容，给出具体的下一步操作：

- 如果有"进行中"的任务：建议继续，说明从哪里接着做
- 如果有明确的功能要开发：建议 `/feat <name>` 开始
- 如果有待讨论的问题：列出问题，和用户讨论
- 如果有 bug 要修：指出相关文件和可能的方向

### 5. 输出格式

```
📋 待办事项
━━━━━━━━━━

<按优先级或分类整理后的内容>

💡 建议下一步：
<具体建议，包含可以直接执行的命令>
```
